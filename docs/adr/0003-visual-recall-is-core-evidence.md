# Visual Recall Is Core Evidence

We treat screenshots, keyframes, and later OCR as required evidence for substantive video reports rather than optional presentation polish. Transcript-only reports miss important visual information in tutorials, software demos, charts, and slides, so phase 1 must support visual recall before the prep/report skill workflow is considered truly usable. The `video-bundle-prep` workflow should first collect metadata, transcript/transcription, comments, retained working audio for subtitle fallback, and a local 1080p-or-best-available working video with `--visual-recall none`; Codex then reads the text evidence, classifies the video, writes `content_profile.json`, and calls `extract-frames` with a chosen visual policy. When only automatic subtitles are available, the bundle keeps the platform transcript as primary, writes a language-aware local transcription comparison transcript, and emits `transcript.comparison.json` so Codex can inspect technical-term disagreements. Local transcription routing should probe actual speech audio first instead of relying only on platform/title language hints. This preserves visual quality while avoiding a rigid rule-based video classifier.
