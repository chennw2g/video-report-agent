# YouTube Visual Recall Downloads A 1080p Working Video

For phase-1 YouTube visual recall, we download a local video-only working file with a 1080p maximum target and keep it under `raw/media/` by default. When subtitles are unavailable, we separately retain working audio for local whisper.cpp transcription rather than assuming the video-only screenshot file has audio. One-step runs can extract screenshots immediately, while skill-led report runs can use `--visual-recall none` first, let Codex classify the video from text evidence, write `visual_selection_plan.json`, then run `extract-frames --plan visual_selection_plan.json` against the retained working video. This favors screenshot quality and repeatable `ffprobe`/`ffmpeg` behavior over direct online stream seeking. Candidate screenshots are plan-guided by default in the skill workflow: `--max-screenshots 0` keeps all coarse baseline and semantic-anchor frames, while skill-side `select-evidence` and `prepare-report --max-images` keep report reading and output manageable.
