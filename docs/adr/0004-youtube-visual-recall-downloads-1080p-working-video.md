# YouTube Visual Recall Downloads A 1080p Working Video

For phase-1 YouTube visual recall, we download a local video-only working file with a 1080p maximum target and keep it under `raw/media/` by default. When subtitles are unavailable, we separately retain working audio for local whisper.cpp transcription rather than assuming the video-only screenshot file has audio. One-step runs can extract screenshots immediately, while skill-led report runs can use `--visual-recall none` first, let Codex classify the video from text evidence, then run `extract-frames` against the retained working video. This favors screenshot quality and repeatable `ffprobe`/`ffmpeg` behavior over direct online stream seeking. Candidate screenshots are uncapped by default (`--max-screenshots 0`) so high-density videos preserve full fixed-interval coverage; skill-side `select-evidence` and `prepare-report --max-images` keep report reading and output manageable.
