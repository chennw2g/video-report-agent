# Xiaohongshu Lightweight Provider First

Xiaohongshu support starts as a lightweight provider boundary rather than importing a full crawler framework. The main project depends on `xhs`, `playwright`, `curl-cffi`, `requests`, and `lxml`; `XHS-Downloader` is checked out under `D:\Workshop\XHS-Downloader` as an external reference/tool and synced with its own uv environment, but it is not vendored or imported by `video-bundle-agent`. `MediaCrawler` is checked out under `D:\W\Codex\external\MediaCrawler` with its own uv environment and is used as the default bounded comment path, but it is not vendored into the main package.

The provider resolves Xiaohongshu URLs and `xhslink.com` short links, reads explicit Netscape cookies from `%APPDATA%\video-bundle-agent\xiaohongshu.cookies.txt` when provided, extracts note metadata/media from the web HTML initial state, writes `media.json`, downloads video/image media, transcribes downloaded videos with whisper.cpp, and runs visual recall against retained video files. Top-level comments are bounded to `--max-comments`, sorted by `like_count`, and use the external MediaCrawler API smoke path by default through an existing CDP Chrome session; raw output is written under `raw/xiaohongshu/mediacrawler/`. The bundled local signer is retained only for explicit fallback/debug use through `--xhs-sign-url local`, `--xhs-sign-url <url>`, or `XHS_SIGN_URL`. Signing and CDP browser state do not replace valid platform permissions: platform account/session risk responses such as `300011` are recorded as `PERMISSION_REQUIRED`; login-expired responses such as `-100` are recorded as `COOKIE_REQUIRED`.

MediaCrawler full crawler workflows, nested comments, search, author-homepage traversal, and login automation remain deferred. The current MediaCrawler use is only for bounded top-level comment recovery and must not become bulk crawling or account-rotation behavior.
