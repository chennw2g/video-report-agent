# 报告视觉规范

状态：v1.2，2026-06-12 更新。
默认视觉模板：Template D / `Editorial Lab`。

本文档只规定报告的视觉表现，包括颜色、版式、字体、间距、卡片、截图呈现、长图和 PDF 适配。本文档不规定报告正文结构、模块顺序、评分位置、标签文案、评论引用规则或具体内容写法；这些内容规则以 `docs/report-output-contract.md` 和 `skills/video-report/SKILL.md` 为准。

## 模板定位

- `Editorial Lab` 是当前默认视觉模板。
- 它适用于 `quick` 和 `deep` 两种报告模式。
- `quick` 和 `deep` 共享同一套视觉系统，区别只体现在内容深度和证据密度。
- 后续可以新增其他视觉模板，但新增模板不应改动现有内容契约。

## 整体气质

- 浅色中文研究简报风格。
- 视觉应克制、清晰、适合长文阅读。
- 不做营销落地页风格。
- 不做高密度后台仪表盘风格。
- HTML 是主输出；正文长图 PNG 是首选分享/汇报成品；PDF 只是可选兼容导出。

## 色彩

- 页面背景：浅冷灰。
- 主内容面板：白色。
- 左侧导航：深蓝灰。
- 主强调色：蓝绿色。
- 次强调色：深绿色。
- 辅助状态色：低饱和琥珀色。
- 警示状态色：低饱和玫红或红色。
- 色彩用途要稳定，不要在同一份报告里频繁切换同一个 token 的含义。

## 字体

- 使用适合中文长文阅读的无衬线字体。
- Windows 默认可使用 `"Segoe UI", "Microsoft YaHei", Arial, sans-serif`。
- 正文字号应适合长时间阅读和 PDF/PNG 输出。
- 行高保持宽松。
- 不使用随视口宽度缩放的字体规则。
- 字距保持默认，不使用负字距。

## 版式

- 桌面端使用左侧导航 + 右侧主阅读区。
- 左侧导航使用深蓝灰背景，宽度稳定。
- 主阅读区居中，最大宽度受控，避免超宽屏下行长过长。
- 顶部使用报告身份区，采用左右分栏：左侧为标题和关键元信息，右侧为封面或代表帧。
- 顶部视觉区默认优先展示平台封面/缩略图；只有内容显式指定 `hero_visual` 或没有可用封面时，
  才使用代表帧。
- 顶部左右分栏应保持视觉平衡，但不强求上下完全平齐；如果强行平齐会导致图片过高、留白或裁切，优先保证整体阅读比例。
- 顶部代表帧与左侧文字块垂直居中对齐即可。
- 顶部标题目标最多两行。标题较长时应在内容生成阶段主动缩短标题；渲染器可自动降低字号以适配两行目标，但不应裁切或隐藏标题文字。
- 顶部标签目标是在桌面宽度下保持一行；如标签过多，应在内容生成阶段减少或缩短标签，而不是通过 CSS 强制遮挡。
- 顶部元信息卡片文字目标不超过两行。频道名、播放量、时长等字段过长时，应优先改短内容；渲染器可自动降低字号以让关键值填进一行，不使用省略号隐藏关键信息。
- 顶部元信息卡片的标签行（平台、频道、发布时间等）应固定在卡片底部区域，不跟随上方数值字号变化而上下漂移。
- 正文使用纵向堆叠的完整宽度模块。
- 每个模块使用白色面板、细边框和轻阴影。
- 模块间距稳定，避免信息块互相挤压。
- 移动端可折叠或隐藏左侧导航，正文改为单列。
- 长图 PNG 输出默认隐藏左侧导航，只截取右侧正文阅读区，并保持 HTML 正文排版。

## 竖屏 Hero 图片

- 顶部 hero 区域需要保护标题和元信息的稳定比例。
- 横屏或接近横屏的封面/代表帧继续按自然比例显示。
- 竖屏视频截图作为 hero 代表图时，使用稳定的 `3:2` 横向容器。
- 竖屏图在 `3:2` 容器内 `contain` 居中显示，两侧允许留白，不裁切主体画面。
- 这条规则只处理顶部 hero 的视觉排版；正文截图是否展示仍由 `docs/report-output-contract.md` 的内容规则决定。

## 面板与卡片

- 默认圆角不超过 8px。
- 面板边框使用低对比度细线。
- 阴影要轻，不做浮夸卡片堆叠。
- 不把大页面区域做成多层嵌套卡片。
- 小卡片只用于局部信息，不用于嵌套整个页面区域。

## 截图与媒体

- 视频截图使用带边框的图片卡片。
- 图片卡片应包含图片区和简短说明区。
- 截图说明的视觉层级低于正文，不喧宾夺主。
- 真实截图应尽量保持可读，不额外加重滤镜。
- 代表帧或封面可用于顶部视觉区。
- 图片尺寸应稳定，避免加载后造成布局大幅跳动。
- 视觉规范只决定截图如何显示；是否应该在正文使用截图由内容契约决定。

## 图表与表格

- 雷达图组件使用简洁线条和低饱和填充。
- 图表颜色遵循主强调色和次强调色。
- 表格使用细分隔线和浅色表头。
- 流程图、矩阵、时间线等图形组件应与整体卡片风格一致。
- 图表不使用装饰性渐变、发光或复杂背景。

## 标签与状态视觉

- 报告可以使用小型 badge 或状态标记。
- badge 使用低饱和底色、短文本空间和稳定圆角。
- badge 可以按信息角色使用不同颜色 token。
- 本文档只规定 badge 的视觉形式，不规定 badge 的具体文字或具体信息分类。
